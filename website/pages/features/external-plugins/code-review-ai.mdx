# code-review-ai

AI-powered architectural review and code quality analysis.

## Overview

| Field | Value |
|-------|-------|
| Marketplace | claude-code-workflows |
| Source | [wshobson/agents](https://github.com/wshobson/agents) |
| License | MIT |
| Default | Pre-selected (recommended) |

## Why Recommended?

This plugin integrates with our [development workflow](/concepts/workflow). The [`/do-review`](/commands/do-review) command uses `code-review-ai:architect-review` as the review agent.

```
┌─────────────────────────────────────────────────────────┐
│  1. SPECIFY                                             │
│     - Define task in CLAUDE.md or /design for complex   │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  2. IMPLEMENT                                           │
│     - Write code                                        │
│     - Write tests                                       │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  3. CODE REVIEW  ◄──── code-review-ai                   │
│     - Run /do-review                                    │
│     - Incorporate feedback                              │
└─────────────────────────────────────────────────────────┘
```

## Capabilities

Reviews cover:

- **Architecture** - patterns, dependencies, design principles
- **Security** - vulnerabilities, auth, input validation
- **Code quality** - complexity, maintainability, test coverage
- **Best practices** - language/framework conventions, performance

## Installation

### Via Installer

```bash
./install.sh        # Fresh install
./install.sh --add  # Add to existing
```

code-review-ai is pre-selected by default.

### Manual

```bash
# Add marketplace
claude plugin marketplace add wshobson/agents

# Install plugin
claude plugin install code-review-ai@claude-code-workflows
```

## Usage

### Via /do-review (recommended)

The [`/do-review`](/commands/do-review) command is the primary way to trigger this plugin:

```
/do-review                    # Review uncommitted changes
/do-review HEAD~3..HEAD       # Review specific commit range
/do-review --branch           # Review current branch vs main
```

It automatically gathers the diff, loads coding standards, and spawns the review agent.

### Direct Request

```
"Review this PR for architectural issues"
"Analyze the security of this module"
"Check this code for best practices"
```

### Via Task Tool

The plugin provides a subagent that can be spawned via the Task tool:

```
subagent_type: "code-review-ai:architect-review"
```

This is what `/do-review` uses under the hood.

## Review Output

The agent provides structured feedback:

1. **Summary** - Overall assessment
2. **Architectural concerns** - Design issues
3. **Security findings** - Vulnerabilities
4. **Code quality** - Improvements
5. **Recommendations** - Actionable next steps

## Integration with Workflow

After implementing a feature:

1. Run `/do-review`
2. Agent reviews the changes
3. Feedback is incorporated
4. Code is committed

This catches issues before they get committed.

See [`/do-review`](/commands/do-review) for details.
