# /claude-code-setup

Check status and update Claude Code Setup from within Claude.

## Usage

```
/claude-code-setup
```

## What It Does

1. **Checks versions**
   - Compares installed version with available version
   - Shows what's new in the update

2. **Offers actions**
   - Upgrade base modules
   - Upgrade custom modules
   - Install new modules (--add)

## Example Output

```
Claude Code Setup Status:

Installed: v22
Available: v23

Changes in v23:
- Decision Log feature
- Context quality improvements

What would you like to do?
1) Upgrade to v23
2) Check custom modules
3) Add new modules
4) Cancel
```

## Upgrade vs Fresh Install

| Action | What happens |
|--------|--------------|
| Upgrade | Preserves User Instructions section |
| Fresh install | Overwrites everything |

Your customizations in the "User Instructions" section of `~/.claude/CLAUDE.md` are preserved during upgrades.

## Modules Requiring API Keys

When installing MCP servers or other modules that require an API key, the interactive prompt cannot work within Claude Code (stdin is consumed by menu interactions).

In this case, Claude will:
1. Install the module with a placeholder
2. Show you the exact configuration with the placeholder
3. Tell you where to get the API key
4. Instruct you to replace the placeholder in `~/.claude.json`

**Example:**
```
Note: MCP "brave-search" requires an API key.

To configure manually, add to ~/.claude.json under "mcpServers":

  "brave-search": {
    "command": "npx",
    "args": ["-y", "@brave/brave-search-mcp-server"],
    "env": {
      "BRAVE_API_KEY": "{{BRAVE_API_KEY}}"
    }
  }

Replace {{BRAVE_API_KEY}} with your key, then restart Claude Code.
```

This applies to any module with `requiresApiKey: true` in its JSON config.
